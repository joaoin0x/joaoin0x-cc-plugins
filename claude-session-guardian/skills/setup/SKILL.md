# /session-guardian:setup

Configuração automática do statusline após instalar o plugin.

## Estratégia (v1.0.5+) — dispatcher pattern

Versões anteriores escreviam o caminho **versionado** (ex: `…/cache/.../1.0.1/statusline/…`) directamente no `statusLine.command`. Cada update do plugin (que cria `…/1.0.X/…` nova) deixava esse caminho stale, obrigando a re-correr setup. UX má.

**v1.0.5 introduz dispatcher**:

1. Setup escreve UM script estável em `$CLAUDE_BASE/session-guardian/statusline-dispatcher.sh` (path NÃO versionado).
2. `statusLine.command` aponta para o dispatcher.
3. O dispatcher, **em runtime**, descobre a versão mais recente em `$CLAUDE_BASE/plugins/cache/joaoin0x-cc-plugins/claude-session-guardian/<X.Y.Z>/` e delega via `exec`.
4. Updates futuros: a próxima sessão arranca, dispatcher escolhe a versão nova automaticamente. **Setup nunca mais é necessário** (a menos que o utilizador queira reverter).

## Procedimento

### PASSO 1 — Descobrir plugin root e script path

```
Bash (single): echo "$CLAUDE_PLUGIN_ROOT"
  → Deve retornar algo como /Users/.../plugins/cache/joaoin0x-cc-plugins/claude-session-guardian/<version>
  → Se vazio: ABORTAR. "CLAUDE_PLUGIN_ROOT não disponível neste contexto. Setup requer invocação como skill do plugin."

STATUSLINE_SCRIPT="$CLAUDE_PLUGIN_ROOT/statusline/session-guardian-statusline.sh"

Bash (single): [ -f "$STATUSLINE_SCRIPT" ] && echo "OK" || echo "MISSING"
  → Se MISSING: ABORTAR. Plugin install possivelmente incompleto.
```

### PASSO 2 — Garantir que o statusline binário é executável

```
Bash (single): chmod +x "$STATUSLINE_SCRIPT"
Bash (single): [ -x "$STATUSLINE_SCRIPT" ] && echo "OK" || echo "FAIL"
```

### PASSO 3 — Detectar settings.json correcto

Tentar por ordem:
1. `$CLAUDE_CONFIG_DIR/settings.json` (se variável definida)
2. `~/.claude-personal/settings.json` (instalação CLDP)
3. `~/.claude/settings.json` (instalação CLDW/default)

```
Bash (single): echo "$CLAUDE_CONFIG_DIR"
  → Se não vazio: SETTINGS_PATH="$CLAUDE_CONFIG_DIR/settings.json"
  → Se vazio: tentar fallbacks por ordem.

Bash (single): [ -f "$SETTINGS_PATH" ] && echo "OK" || echo "MISSING"
  → Se MISSING: ABORTAR.
```

### PASSO 4 — Criar directório de state e dispatcher

```
Bash (single): CLAUDE_BASE="${CLAUDE_CONFIG_DIR:-$HOME/.claude}"
Bash (single): STATE_DIR="$CLAUDE_BASE/session-guardian"
Bash (single): mkdir -p "$STATE_DIR/checkpoints"
Bash (single): chmod 0700 "$STATE_DIR"

DISPATCHER_PATH="$STATE_DIR/statusline-dispatcher.sh"
```

### PASSO 5 — Gerar dispatcher script

**OBRIGATÓRIO — usar Write TOOL, NÃO heredoc shell. Conteúdo abaixo é literal:**

Write TOOL no path `$DISPATCHER_PATH` com o conteúdo exacto seguinte (substituir `__CLAUDE_BASE__` pelo valor real apurado no PASSO 4):

```bash
#!/bin/bash
# claude-session-guardian — Statusline dispatcher
# Generated by /session-guardian:setup. Do NOT edit by hand — re-run setup.
#
# PURPOSE: Resolve and exec the latest installed plugin statusline at
# runtime. Avoids hardcoding version-specific paths in settings.json,
# which would otherwise break on every plugin update.
#
# RESOLUTION ORDER:
#   1. Highest semver subdirectory of <CLAUDE_BASE>/plugins/cache/<marketplace>/<plugin>/
#   2. Fallback: marketplace tree (live source if Claude Code uses path-based install)
#   3. If neither resolves: drain stdin, emit minimal model name, log error.

set -u

CLAUDE_BASE="${CLAUDE_CONFIG_DIR:-__CLAUDE_BASE__}"
MARKETPLACE="joaoin0x-cc-plugins"
PLUGIN="claude-session-guardian"

CACHE_ROOT="$CLAUDE_BASE/plugins/cache/$MARKETPLACE/$PLUGIN"
MARKET_ROOT="$CLAUDE_BASE/plugins/marketplaces/$MARKETPLACE/$PLUGIN"
ERR_LOG="$CLAUDE_BASE/session-guardian/dispatcher-errors.log"

resolve_latest() {
    local root="$1"
    [ -d "$root" ] || return 1
    local v
    v=$(ls -1 "$root" 2>/dev/null | grep -E '^[0-9]+\.[0-9]+\.[0-9]+$' | sort -V | tail -1)
    [ -z "$v" ] && return 1
    local bin="$root/$v/statusline/session-guardian-statusline.sh"
    [ -x "$bin" ] || return 1
    printf '%s' "$bin"
}

LATEST_BIN=$(resolve_latest "$CACHE_ROOT")
if [ -z "$LATEST_BIN" ]; then
    # Marketplace tree (path install) is unversioned, points directly at script
    CANDIDATE="$MARKET_ROOT/statusline/session-guardian-statusline.sh"
    [ -x "$CANDIDATE" ] && LATEST_BIN="$CANDIDATE"
fi

if [ -z "$LATEST_BIN" ]; then
    mkdir -p "$(dirname "$ERR_LOG")" 2>/dev/null
    echo "[$(date -u +%FT%TZ)] dispatcher: no installed statusline found in $CACHE_ROOT or $MARKET_ROOT" >> "$ERR_LOG" 2>/dev/null

    INPUT=$(cat)
    MODEL=$(printf '%s' "$INPUT" | jq -r '.model.display_name // .model.name // "Claude"' 2>/dev/null)
    printf '%s\n' "$MODEL"
    exit 0
fi

exec "$LATEST_BIN"
```

Após escrever o ficheiro:

```
Bash (single): chmod +x "$DISPATCHER_PATH"
Bash (single): [ -x "$DISPATCHER_PATH" ] && echo "OK" || echo "FAIL"
```

Verificar resolução (sanity test do dispatcher):

```
Bash (single): bash -n "$DISPATCHER_PATH" && echo "SYNTAX OK" || echo "SYNTAX FAIL"
  → Se FAIL: ABORTAR, restaurar backup.
```

### PASSO 6 — Backup atómico do settings.json

```
BACKUP_PATH="${SETTINGS_PATH}.pre-session-guardian"

Bash (single): cp "$SETTINGS_PATH" "$BACKUP_PATH"
Bash (single): sync
Bash (single): [ -f "$BACKUP_PATH" ] && echo "OK" || echo "FAIL"
  → Se FAIL: ABORTAR (limpar dispatcher do PASSO 5).
```

### PASSO 7 — Mutação atómica do settings.json (apontar para dispatcher + refreshInterval)

**OBRIGATÓRIO — usar jq, nunca sed/awk sobre JSON.**

`refreshInterval: 60` (NOVO em v1.1.0) força o statusline a re-executar a cada 60s mesmo quando a sessão está idle, garantindo que `rate-state.json` é actualizado com regularidade. Sem isto, em sessões idle o ficheiro pode ficar stale e o monitor lê valores antigos. Suportado em CLI 2.1.121+.

```
Bash (single): TMP="${SETTINGS_PATH}.tmp.$$"

Bash (single): jq --arg cmd "$DISPATCHER_PATH" '.statusLine = {type: "command", command: $cmd, padding: 0, refreshInterval: 60}' "$SETTINGS_PATH" > "$TMP"

Bash (single): [ -s "$TMP" ] && echo "OK" || echo "EMPTY"
  → Se EMPTY: rm $TMP, restaurar backup, ABORTAR.

Bash (single): jq empty "$TMP" 2>/dev/null && echo "VALID" || echo "INVALID"
  → Se INVALID: rm $TMP, restaurar backup, ABORTAR.

Bash (single): chmod 0600 "$TMP"
Bash (single): mv -f "$TMP" "$SETTINGS_PATH"
```

### PASSO 8 — Validação pós-escrita

```
Bash (single): jq -r '.statusLine.command' "$SETTINGS_PATH"
  → Deve retornar exactamente $DISPATCHER_PATH.
  → Se não bater: restaurar backup, reportar erro.
```

### PASSO 8.5 — Health check do monitor (NOVO v1.1.0)

Validar que o detector `monitors/watch-rate-state.sh` é sintacticamente válido e executa pelo menos uma iteração sem crashar. Não testa o flow completo (que requer mudanças em rate-state ao longo do tempo) — apenas confirma que o script arranca limpo.

```
MONITOR_SCRIPT="$CLAUDE_PLUGIN_ROOT/monitors/watch-rate-state.sh"

Bash (single): [ -f "$MONITOR_SCRIPT" ] && echo "PRESENT" || echo "MISSING"
  → Se MISSING: warning ao user (não abortar — fallback /loop legacy continua viável).
  → Saltar para PASSO 9.

Bash (single): bash -n "$MONITOR_SCRIPT" && echo "SYNTAX_OK" || echo "SYNTAX_FAIL"
  → Se SYNTAX_FAIL: ABORTAR setup, restaurar backup, reportar erro.

Bash (single): chmod +x "$MONITOR_SCRIPT"

Sanity run (3s timeout):
  Bash (single): bash "$MONITOR_SCRIPT" > /tmp/.guardian-monitor-test.$$ 2>&1 & PID=$!; sleep 3; kill $PID 2>/dev/null; wait $PID 2>/dev/null

Bash (single): [ -s /tmp/.guardian-monitor-test.$$ ] || [ -f /tmp/.guardian-monitor-test.$$ ] && echo "RAN" || echo "NORUN"
Bash (single): rm -f /tmp/.guardian-monitor-test.$$
  → Se NORUN: warning (script existe mas não correu — pode ser problema de permissões).
```

### PASSO 9 — Health check end-to-end (CRÍTICO)

Health check tem agora dois objectivos:
1. Confirmar que o dispatcher resolve para uma versão funcional do statusline
2. Confirmar que essa versão aceita ambos os formatos de `resets_at` (epoch e ISO-8601)

Importante: o teste invoca o **dispatcher**, não o `STATUSLINE_SCRIPT` directamente. Isto valida toda a cadeia, simulando o que o Claude Code real fará.

```
Bash (single): TEST_DIR="$CLAUDE_BASE/session-guardian"
Bash (single): rm -f "$TEST_DIR/rate-state.json"

# Health check 1: Unix epoch (formato actual do Claude Code)
Bash (single): echo '{"model":{"display_name":"healthcheck"},"rate_limits":{"five_hour":{"used_percentage":42,"resets_at":1777082400}}}' | "$DISPATCHER_PATH" >/dev/null

Bash (single): [ -f "$TEST_DIR/rate-state.json" ] && jq -e '.used_percentage_5h == 42 and (.resets_at_5h | startswith("2026"))' "$TEST_DIR/rate-state.json" >/dev/null && echo "PASS" || echo "FAIL"
  → Se FAIL: ABORTAR setup. Restaurar backup do settings.json. Apagar dispatcher.
    Reportar: "Health check 1 (epoch via dispatcher) falhou. Versão actual do plugin não aceita Unix epoch ou dispatcher não resolveu para versão correcta. Logs: $TEST_DIR/statusline-errors.log e $TEST_DIR/dispatcher-errors.log."

# Health check 2: ISO-8601
Bash (single): rm -f "$TEST_DIR/rate-state.json"
Bash (single): echo '{"model":{"display_name":"healthcheck"},"rate_limits":{"five_hour":{"used_percentage":58,"resets_at":"2026-04-30T12:00:00Z"}}}' | "$DISPATCHER_PATH" >/dev/null

Bash (single): [ -f "$TEST_DIR/rate-state.json" ] && jq -e '.used_percentage_5h == 58' "$TEST_DIR/rate-state.json" >/dev/null && echo "PASS" || echo "FAIL"
  → Se FAIL: ABORTAR setup. Restaurar backup. Apagar dispatcher.

Bash (single): rm -f "$TEST_DIR/rate-state.json"
```

Se ambos os health checks passam, prosseguir para PASSO 10.

### PASSO 10 — Reportar versão resolvida ao utilizador

```
Bash (single): RESOLVED=$(grep -oE '/[^"]+statusline/session-guardian-statusline\.sh' <<< "$(bash -c 'set -x; "'"$DISPATCHER_PATH"'" </dev/null' 2>&1 || true)" | head -1)
```

(O dispatcher faz `exec`, não imprime path; mas podemos detectar versão pela mtime ou inspeccionar o cache root directamente.)

Mais simples: listar versões disponíveis e indicar a mais recente:

```
Bash (single): ls -1 "$CLAUDE_BASE/plugins/cache/joaoin0x-cc-plugins/claude-session-guardian/" 2>/dev/null | sort -V | tail -1
```

## Restauro em caso de falha

Se qualquer passo 5-9 falha após escrever o dispatcher:

```
Bash (single): cp "$BACKUP_PATH" "$SETTINGS_PATH"
Bash (single): rm -f "$DISPATCHER_PATH"
Bash (single): rm -f "${SETTINGS_PATH}.tmp.$$"

Emitir erro com detalhe do passo que falhou.
```

## Notas de segurança

- Nunca aceitar path para `settings.json` que contenha `..` ou que não esteja dentro de `$HOME`.
- Validar que `$CLAUDE_PLUGIN_ROOT` é um path absoluto existente antes de o usar.
- Permissões 0600 no settings.json resultante (preservar permissões originais se mais restritivas).
- Dispatcher é executável e contém apenas strings literais geradas por este setup; nunca interpola dados do utilizador além do `$CLAUDE_CONFIG_DIR` resolvido. Se essa variável fosse maliciosa (improvável), a única consequência seria o dispatcher falhar a resolver — comportamento fail-safe.
